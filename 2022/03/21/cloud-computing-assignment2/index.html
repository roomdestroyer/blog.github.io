云计算课程 作业2

2-1、列举威胁作用者有哪些，并简要说明。
  • 匿名攻击者：是云中没有权限的、不被信任的云服务用户，通常是一个外部软件程序，通过公网发动网络攻击。
  • 恶意服务作用者：能截取并转发云内的网络流量，通常是带有被损害的或恶意逻辑的服务代理（或伪装成服务代理的程序），也有可能是能够远程截取并破坏消息内容的外部程序。
  • 授信的攻击者：与同一云环境中的云用户共享 IT 资源，试图滥用合法的证书或挪用敏感和保密的信息，来把云提供者以及与他们共享 IT 资源的云租户作为攻击目标。
  • 恶意的内部人员：人为的威胁作用者，通常是现任或前任雇员，或是能够访问云提供者资源范围的第三方，试图滥用对云资源的访问特权来破坏 IT 资源。
2-2、列举云安全威胁有哪些，并简要说明。
  • 流量窃听：当数据在传输到云中或在云内部传输时（通常是从云用户到云提供者）被恶意的服务作用者被动地截获，用于非法信息收集的目的。
  • 恶意媒介：消息被恶意服务作用者截获并且被篡改，因此可能会破坏消息的保密性和完整性，还有可能在把消息转发到目的地之前插入有害的数据。
  • 拒绝服务：攻击者发送多个云服务请求，使云服务器在内存、处理性能、网络、信道等方面的 IT 资源过载而无法正常运行，成功的 DoS 攻击使得服务器性能恶化或失效。
  • 授权不足：错误地授予了攻击者访问权限或是授权太宽泛，导致攻击者能够访问到本应该受到保护的 IT 资源，如果使用弱密码或共享账户来保护 IT 资源，就可能受到这种攻击。
  • 虚拟化攻击：利用虚拟化平台中的漏洞来危害虚拟化平台的保密性、完整性和可用性，例如，一个授信的攻击者成功接入了一台虚拟服务器，破坏了它底层的物理服务器，而一个物理服务器可能要向多个云用户提供虚拟化 IT 资源，这样会导致其它用户的 IT 资源受损。
  • 信任边界重叠：恶意的云服务用户可以把目标设定为共享的 IT 资源，意图损害其他共享同样信任边界的云服务用户或 IT 资源，结果就是其它云服务用户都受到攻击的影响。
